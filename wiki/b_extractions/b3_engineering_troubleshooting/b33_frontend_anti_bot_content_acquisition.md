# 强前端反爬内容获取

## 提取结果

- 对强前端站点，最有价值的抓取对象通常是渲染后的 DOM 或正文文本，而不是原始 HTML
- `curl`、公开 API 或普通 headless dump 失败，不等于真实浏览器匿名访问也失败
- 验证是否抓到正文时，必须同时看标题、正文、页面结构标记

## 稳定处理顺序

1. 区分“内容不存在”和“抓取方式被拦”
2. 先试简单抓取
3. 不行再切到真实浏览器
4. 从页面上下文直接取文
5. 做三重验证
