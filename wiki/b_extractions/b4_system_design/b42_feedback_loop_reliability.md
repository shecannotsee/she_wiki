# 反馈链路可靠性

## 提取结果

- 关键系统不能只监控业务执行结果，还要监控承担反馈职责的监控链路本身
- 当一个监控承担决策修正作用时，它已经是系统骨架的一部分，必须具备存活性检查和可验证性
- 元监控的设计目标不是无限叠层，而是在成本可接受的前提下，把关键反馈链路加固到足够可靠

## 核心结构

1. 先识别哪个环节承担关键反馈职责
2. 再区分业务失败与反馈链路失效
3. 对关键监控补上存活性检查，例如 `heartbeat`
4. 对关键监控补上历史记录、检索和回查能力
5. 按 ROI 决定是否继续上叠，而不是机械追求更高层级

## 可直接复用

- 不把告警系统默认当成可靠事实来源，先问它有没有被验证过
- 对关键监控同时设计“活着”与“可审计”两类保证
- 二级监控通常已经能覆盖大部分反馈链路失效问题，继续叠加前先确认真实受众和收益
