# 线程池的最小系统骨架

## 提取结果

- 线程池的核心价值不是“开更多线程”，而是把线程创建销毁成本从任务执行路径里移出去
- 一个可用线程池至少要同时解决四件事：任务提交、结果返回、空闲休眠、生命周期控制
- 线程池设计里，任务模型和停机语义必须先定义清楚，否则实现很容易只剩一个“能跑任务的循环”
- `terminate` 和 `cancel` 这种语义区分很关键：一个强调平滑停机，一个强调直接放弃未处理任务
- 线程安全队列通常不是线程池的附属细节，而是系统吞吐和控制语义的核心部件

## 核心结构

1. 用 worker 集合承载可复用执行资源
2. 用线程安全任务队列接收外部任务
3. 用休眠唤醒机制避免空转浪费 CPU
4. 用 future 类机制把执行结果交还给提交者
5. 用显式状态位控制启动、停机、取消和后续扩展能力

## 可直接复用

- 设计线程池时，先写清楚“是否继续接单”“是否清空存量任务”“如何返回结果”这三件事
- 不要把线程池简化成“循环从队列拿任务执行”，还要明确外部观察接口和状态语义
- 需要平滑退出时，保留已接收任务并等待 worker 消化；需要快速中止时，再显式丢弃剩余任务
- 如果任务提交者需要结果或异常，就不要只暴露 fire-and-forget 接口
- 队列实现先求语义正确，再决定是否为高频场景引入细粒度锁或无锁结构
