# 按认知任务分层的软件文档

## 提取结果

- 软件文档不是单一类型内容，而应按读者当前的认知任务分成不同文档类型
- 最稳定的四类职责是：
  - 教程：帮助新手通过实践入门
  - 操作指南：解决具体任务
  - 参考手册：准确描述系统与接口
  - 解释性文档：提供背景、原因和取舍
- 文档质量差，常常不是因为内容不足，而是因为不同职责被混写在同一页面里
- 文档结构有自然塌缩倾向，因为相邻类型表面上很像；只有显式分层，作者和读者才能都获得清晰边界
- 好文档治理首先是按学习、做事、查阅、理解这四种需求分层，而不是按作者想到什么就写什么

## 稳定结构

1. 先判断当前内容是在服务学习、做事、查阅还是理解
2. 再决定它应该写成教程、操作指南、参考手册还是解释性文档
3. 每类文档只承担一个主职责，避免混入相邻类型内容
4. 用显式栏目或信息架构把四类内容分开组织
5. 持续检查文档是否因为边界模糊而重新塌缩

## 可直接复用

- 写文档前先问：这页是在教新手入门，还是在帮熟手完成任务，还是在供人查参数，还是在解释背景
- 如果一页同时在讲步骤、概念、API 和设计取舍，通常说明它需要拆分
- 教程先保证新手能做成事，不要急着塞入最佳实践和大段原理
- 操作指南优先回答 “How do I ...”，不要把它写成微型教程或微型百科
- 参考手册按代码和对象结构组织，重点是准确、一致、可检索
- 解释性文档用来讲为什么，而不是讲下一步点哪里
