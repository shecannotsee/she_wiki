# 知乎回答：监控的监控

## 记录时间

- 2026-04-08 CST

## 原始讨论对象

- 原始来源链接：https://www.zhihu.com/question/589964007/answer/3584445924
- 可访问参照页：https://my.oschina.net/liyanqing/blog/15207304

## 原始信息

- 文章主线：
  - 组织里的决策和执行天然存在信息差与执行误差，因此需要监控作为反馈链路
  - 监控本身也会失效，所以关键监控还需要更上一层的“监控的监控”
- 文中用于说明问题的业务例子：
  - 存储系统为每个用户设置 `quota`
  - 当用户配额接近上限时，一级监控负责发出告警，避免长任务因写入失败而中断
  - 但一级监控可能因为程序未启动、程序 bug、采集失败、消息发送失败等原因失效
- 文中给出的加固抓手：
  1. 用 `heartbeat` 检查关键监控程序是否正常启动和持续运行
  2. 保留历史记录，使监控行为可检索、可回查、可验证
  3. 对关键心跳机制本身再做保底检查，降低单点失效风险
- 文中对层级边界的判断：
  - 一级监控主要服务执行层，修正执行策略
  - 二级监控主要服务决策层，保证反馈链路本身可靠
  - 更高层级并非不能做，而是要按 ROI 判断，通常缺乏明确目标用户，边际价值较低

## 原始结论

- 关键系统里，最危险的不是“没有监控”，而是“把不可靠的监控当成可靠反馈”
- 对关键反馈环节，应该把“监控是否工作正常”本身也纳入系统设计
- 元监控的价值不在于无限叠层，而在于用有限成本把关键反馈链路加固到足够可靠
